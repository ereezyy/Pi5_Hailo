## 2024-05-18 - Path Traversal Vulnerability in Hailo Service
**Vulnerability:** The Hailo service (`hailo_service.py`) accepted the `hef_path` parameter directly from incoming requests to load machine learning models without validating if the path was within allowed directories. This could allow an attacker to potentially read arbitrary files by passing a path like `../../../../etc/passwd` to the API.
**Learning:** File paths supplied by users or APIs must be securely validated before they are used to open or execute files. Simple string matching is insufficient. Using `os.path.realpath` resolves symlinks and canonicalizes the path, while `os.path.commonpath` accurately confirms if the target path resides securely within the allowed parent directory.
**Prevention:** Implement a strict allowlist of directories for model loading. Always use `os.path.realpath` to resolve both the target file path and the allowed directories, and verify using `os.path.commonpath([allowed_dir, real_target_path]) == allowed_dir`.
